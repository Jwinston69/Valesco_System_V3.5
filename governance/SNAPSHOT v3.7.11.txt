RUNTIME PHASE MODEL

Settled runtime phase model:

CE retrieval
-> Router
-> Architect
-> Validator
-> Estimator Runtime decision adapter
-> Merge Agent
-> post-merge quantity/pricing

RUNTIME MODE CLASSIFICATION

Deterministic interface REPL:

run_valesco.py
bin/run_valesco_portable.bat

This mode uses mock/static CE retrieval before entering the shared kernel
boundary.

Quantity/pricing execution mode:

engine/scripts/mvp_runner_v2_2.py

This mode uses the CE backend command harness before entering the shared kernel
boundary and executes explicit quantity/pricing commands.

The two runtime modes share the kernel boundary but are not a single merged
runtime.

KERNEL BOUNDARY POSITION

Kernel decision chain:

Router
-> Architect
-> Validator

Estimator Runtime:

Decision adapter only. It is not pricing authority and is not kernel.

Merge Agent:

State integration / kernel output boundary.

Quantity and pricing:

Post-merge extensions only.

CE acquisition:

Pre-kernel only.
